# ADR-003: Use huh.NewSelect for the home menu, looping until Quit

## Status

Accepted

## Date

2026-06-20

## Context

The home menu needs an interactive selection list under the banner on the bare `rc` command. The codebase already uses `charm.land/huh/v2` throughout the CLI layer (`internal/cli/setup.go`, `add.go`) with shared themes in `internal/cli/theme.go`, while `charm.land/bubbletea/v2` is reserved for the rich runtime UI in `internal/core/run/ui`. The PRD (ADR-001) listed a number quick-select shortcut alongside arrow/j-k/Enter/Esc navigation. The PRD also requires returning to the menu after each action until the user quits.

## Decision

Build the menu with `huh.NewSelect[string]()`, following the existing pattern in `resolveSkillSelection`/`resolveAgentSelection`, themed via the existing `darkHuhTheme()`. Navigation uses huh's native bindings (arrow keys and j/k, Enter to confirm, Esc/Ctrl-C to abort). The bare-command handler renders the banner, then loops: present the select, dispatch the chosen action, and re-present the select; selecting "Quit" or aborting (Esc/Ctrl-C, surfaced as huh's user-abort error) exits the loop cleanly.

Drop the **number quick-select** shortcut from the MVP — huh's select does not provide it natively, and adding it would require a custom Bubble Tea model. This is an intentional, accepted reduction from ADR-001's wish list.

## Alternatives Considered

### Alternative 1: Custom Bubble Tea model

- **Description**: Hand-write a Bubble Tea list model in the CLI layer to support arrows + j/k + Esc/q + number quick-select exactly as ADR-001 described.
- **Pros**: Full control over keybindings, including number quick-select.
- **Cons**: Introduces a new TUI model in a layer that otherwise uses huh; more code to maintain; diverges from the established CLI pattern.
- **Why rejected**: YAGNI and consistency — the number shortcut is low value at five items and not worth a bespoke model.

## Consequences

### Positive

- Consistent with the established CLI interaction pattern and brand theme.
- Minimal new code; reuses huh and existing themes.

### Negative

- No number quick-select; users navigate with arrows/j-k and Enter.

### Risks

- huh's select aborts (Esc/Ctrl-C) must be mapped to a clean exit rather than an error surfaced to the user. Mitigation: treat the user-abort error as "Quit".

## Implementation Notes

The loop lives in the bare-command `RunE` in `internal/cli/root.go`. Each iteration constructs a fresh `huh` form.
