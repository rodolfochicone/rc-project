# ADR-003: Keep the `rc-` name prefix; plugin is an additive Claude-only channel

- **Status:** Accepted
- **Date:** 2026-06-25

## Context

Plugin skills are namespaced as `plugin-name:skill-name`, so the `rc` plugin's skills are
invoked as `rc:<skill-name>`. The bundled skills already carry a `rc-` prefix in their
directory names (e.g. `rc-execute-task`), which would render as `rc:rc-execute-task` under
the plugin namespace. Renaming the directories to drop the prefix would change what
`rc setup` installs (`/rc-execute-task`), require rewriting cross-references in ~18
`SKILL.md` files, and invalidate existing user memory and documentation.

Separately, Claude Code plugins serve Claude Code only; RC also targets codex, cursor,
droid, gemini, opencode, and pi, which do not consume the plugin format.

## Decision

- **Keep the `rc-` prefix** on all skill/command names. The plugin exposes them as
  `rc:rc-*`; `rc setup` continues to expose them as `/rc-*`. No directory renames, no
  cross-reference edits, no naming change to `rc setup`.
- **Scope the plugin as an additive, Claude-only channel.** `rc setup` and `rc sync`
  remain unchanged and are the path for all non-Claude agents. A Claude user picks one
  channel (plugin OR setup) to avoid visually redundant listings; the two cannot conflict
  technically because plugin skills are namespaced.

## Alternatives Considered

- **Drop the `rc-` prefix** for cleaner `rc:execute-task` invocation: rejected — broad,
  risky change touching `rc setup` naming, every SKILL.md cross-reference, docs, and user
  memory, for a cosmetic gain. Violates surgical-change principle.
- **Replace `rc setup` with the plugin for Claude** (auto-skip skills when plugin
  detected): rejected for this iteration — adds detection logic to `setup`; deferred until
  the plugin channel is proven.

## Consequences

- **Positive:** Surgical change; `rc setup`, cross-references, and memory all keep working.
- **Negative / accepted trade-off:** Slightly verbose invocation (`/rc:rc-execute-task`).
- **Negative / accepted trade-off:** A user running both `rc setup` and the plugin sees
  each skill twice (once per channel). Documented; mitigated by recommending one channel.
