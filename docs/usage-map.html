rc · usage map

Idea in, pull request out — every phase a skill, every artifact markdown.

Slash commands are the entry points. Each command drives one or more skills, heavy phases delegate to model-pinned agents, hooks guard everything in the background, and the memory skills make the next run smarter than the last. Artifacts live under .rc/tasks/<slug>/ — stop anywhere, edit by hand, resume.

The pipeline

Run it phase by phase, or hand the whole line to /rc-pipe <idea>.

1 · Plan /rc-plan

PRD → TechSpec → task files. Skills rc-create-prd, rc-create-techspec, rc-create-tasks; agents rc-prd, rc-techspec, rc-tasks.

2 · Execute /rc-exec

Each task implemented, verified, tracked. Skill rc-execute-task; agents rc-exec (hard) / rc-exec-bulk (parallel).

3 · Quality /rc-gan

Only for subjective surfaces (UI, CLI, copy): adversarial generator↔evaluator loop up to a target score.

4 · Review /rc-review

One simplify pass, then ≤3 rounds of rc-review-round + rc-fix-reviews until a round is clean.

5 · Docs /rc-docs

README, OpenAPI, Postman — regenerated from the real code. (Claude-only command; the skills run anywhere.)

6 · Ship /rc-git

Feature branch → push → PR, confirming each outward-facing step and the PR target.

7 · Learn rc-instincts

Distills the session's corrections into confidence-scored instincts under .rc/instincts/.

OpenCode reaches phases 1–4 through per-phase commands too: /rc-prd · /rc-techspec · /rc-tasks · /rc-fix. On Claude Code the same phases are the skills /rc:rc-create-prd, /rc:rc-create-techspec, /rc:rc-create-tasks.

The five layers

Commands

Entry points. Claude: /rc-plan /rc-exec /rc-pipe /rc-review /rc-git /rc-gan /rc-docs. OpenCode adds per-phase /rc-prd /rc-techspec /rc-tasks /rc-fix.

Skills

The workflow logic — 31 SKILL.md files commands and agents route into. Skills own the artifacts under .rc/tasks/<slug>/.

Agents

One per phase, model pinned to the job: rc-prd/rc-techspec/rc-exec/rc-review/rc-fix/rc-gan on opus, rc/rc-tasks/rc-exec-bulk/rc-librarian on sonnet, rc-git/rc-explorer on haiku. Explorer and librarian are read-only on both runtimes.

Hooks

Always-on guards: git-guard, commit-guard, go-mod-guard, gateguard block before damage; go-fmt and observe run after edits; session-recall, phase-reminder, precompact-capture, notify keep sessions oriented (Claude-side; OpenCode mirrors the guards via rc-hooks.ts).

Memory

What makes run N+1 better: rc-project-memory (facts), rc-workflow-memory (run context), rc-instincts (trigger→action patterns), rc-reflect (what asset to build next).

A typical day

  1. New feature — /rc-plan payment-retries; answer the PRD/TechSpec questions; skim the generated task files.
  2. Build — /rc-exec payment-retries; tasks are implemented, verified and tracked while the guard hooks watch git and edits.
  3. Harden — /rc-review payment-retries loops simplify → review → fix until a round comes back clean.
  4. Ship — /rc-git branches, pushes and opens the PR, confirming each outward-facing step; rc-instincts captures what the session taught.
  5. Anytime — rc-explorer for lookups, rc-analyze when something breaks, rc-doctor when rc itself misbehaves.
