Mapa de Skills & Agentes

Qual ferramenta puxar em cada situação. Skills você invoca inline (/nome ou via Skill) para carregar know-how no seu próprio contexto. Agentes são subagentes disparados em paralelo que devolvem só a conclusão. Leia de cima pra baixo pela situação, não pelo nome.

00

O contrato do projeto

No rc, essas skills não são opcionais — pular qualquer uma invalida a tarefa.

Sempre, antes de fechar

  • Antes de escrever Go → golang-pro
  • Antes de corrigir bug → systematic-debugging + no-workarounds
  • Antes de escrever/mudar teste → testing-anti-patterns
  • Antes de declarar QUALQUER tarefa pronta → rc-final-verify e make verify a 100% (fmt · lint · test · build)
  • make lint tem tolerância zero. make verify só passa no WSL (no Windows quebra em SQLite/CRLF/cgo).

02

Por domínio — o que puxar

Cada card é uma situação. Dentro, as ferramentas ordenadas por quando pegar cada uma.

Go & Runtime

escrevendo Go
golang-pro obrig
Qualquer código Go: goroutines/channels, generics, interfaces, gRPC, CLI, benchmarks, table-tests.
go-reviewer agent
Review de mudança em Go: idioma, concorrência, erros, performance. Dispare após escrever.
go-build-resolver agent
Build/vet/lint Go quebrou — conserta com diff mínimo, sem redesenho.
bubbletea · tui-glamorous skill
TUI em Go (Bubble Tea / Lip Gloss / Gum), layouts, spinners, kanban.

Debug & Correção

bug, falha, crash
systematic-debugging obrig
Todo bug/teste falhando/comportamento inesperado — antes de propor fix. Causa-raiz, não sintoma.
no-workarounds obrig
Bloqueia gambiarra: type-assertion, lint-suppress, error-swallow, timing-hack, monkey-patch.
silent-failure-hunter agent
Caça erros engolidos, fallbacks ruins e propagação de erro faltando.
build-error-resolver agent
Build/TypeScript quebrado — só o necessário pra ficar verde.

Testes & QA

testar, validar branch
testing-anti-patterns obrig
Antes de escrever/mudar teste ou mock — evita testar mock e poluir produção.
qa-execution skill
QA de projeto inteiro como usuário real: build+lint+test+startup, fluxos E2E, fix causa-raiz, re-roda o gate.
qa-report skill
Planejar QA antes de executar: test plans, casos, regressão, bug reports.
tdd-guide agent
Escrever teste primeiro, feature depois. 80%+ de cobertura.
pr-test-analyzer agent
Avalia qualidade/cobertura de testes de um PR (cobertura comportamental).
e2e-runner · vitest agent
E2E (Agent Browser/Playwright) e testes de frontend.

Code Review

após escrever código
/code-review · /simplify skill
Diff atual: bugs de correção (code-review) ou limpeza/simplificação (simplify). Use ultra p/ review multi-agente na nuvem.
code-reviewer agent
Review geral de qualidade/segurança/manutenção. Dispare logo após mudar código.
*-reviewer agent
Por linguagem: go / typescript / svelte / vue. Dispare no arquivo certo.
adversarial-review skill
Revisor no modelo oposto (Claude→Codex) desafiando por lentes distintas.
review-squad · review-pipeline skill
Esquadrão paralelo roteado por linguagem, veredito único.
comment-analyzer · type-design-analyzer agent
Lentes específicas: rot de comentário; design de tipos/invariantes.

Arquitetura & Plano

antes de construir
brainstorming obrig
Antes de QUALQUER trabalho criativo (feature/componente). Explora intenção e requisitos.
council skill
Debate multi-conselheiro em decisão de alto impacto (arquitetura/tech/produto). Preserva dissenso.
architectural-analysis skill
Auditoria: dead code, duplicação, anti-padrões, type confusion, code smell.
architect · code-architect agent
Desenho de sistema / blueprint de feature a partir dos padrões do repo.
planner · Plan agent
Plano passo-a-passo, arquivos críticos, trade-offs. Antes de refactor grande.
code-explorer · Explore agent
Mapear feature/arquitetura existente e caminhos de execução (read-only fan-out).

Refactor & Performance

Catálogo Fowler de code smells → relatório priorizado em docs/_refacs/.

Gargalos, memory leaks, bundle, render. Profiling e melhoria algorítmica.
refactor-cleaner agent
Dead code / duplicatas: knip, depcheck, ts-prune e remoção segura.
code-simplifier agent
Clareza/consistência preservando comportamento; foco no que mudou.
